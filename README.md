Проект представляет собой консольное клиент–серверное приложение на C++, демонстрирующее межпроцессное взаимодействие через именованные каналы (Named Pipes) в Windows. Сервер хранит массив сотрудников (Employee) и обрабатывает запросы нескольких клиентов одновременно, каждый из которых обслуживается в отдельном потоке. Клиенты формируют команды для чтения (READ) и записи (WRITE) сотрудников и получают результаты через бинарный протокол (CommandPacket).

Данные сотрудников представлены структурой Employee, содержащей номер, имя и количество отработанных часов. Логика работы с ними инкапсулирована в модуле EmployeeStorage, который реализует методы поиска (Find) и обновления (Update). Для синхронизации доступа к общему хранилищу используются WinAPI-объекты Mutex и Semaphore.

Проект реализован на стандарте C++17 и использует WinAPI для работы с потоками, именованными каналами и объектами синхронизации (CreateNamedPipe, ConnectNamedPipe, WriteFile, ReadFile, CreateThread). Сборка осуществляется с помощью CMake, настроенного под компилятор MSVC, структура проекта модульная: отдельные файлы для сервера, клиента, хранилища и общих структур (employee.h, protocol.h).

Обработка ошибок включает контроль корректности открытия канала, чтения и записи данных, а также проверку корректности вводимых пользователем номеров сотрудников и команд. Для тестирования логики работы с данными и механизмов синхронизации применяются юнит-тесты на базе GoogleTest.
